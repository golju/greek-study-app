# Промт: приложение для изучения греческих слов

## Контекст

Я учу греческий язык. У меня уже есть готовая база глаголов с переводами и
формами времён в файле `docs/greek-words.json`. Нужно собрать **простое
веб-приложение** (одна HTML-страница), в котором я смогу учить эти слова.

Дизайн **не важен**: можно совсем без стилей, голый HTML + минимум CSS для
читаемости. Главное — функциональность и скорость работы.

## Стек

- Чистый статический сайт: один `index.html` в корне репозитория.
- Без сборщиков, без npm, без фреймворков.
- Vanilla JS (можно вынести в `assets/js/app.js`).
- Минимум CSS (можно вынести в `assets/styles/main.css`)
- Данные читаются из `docs/greek-words.json` через `fetch()`.
- Должно работать при открытии через локальный статический сервер
  (`python3 -m http.server`). Открытие через `file://` не требуется.

## Источник данных

Файл: `docs/greek-words.json`. Это массив объектов вида:

```json
{
  "id": "agapao",
  "verb": "αγαπάω",
  "translation": "любить",
  "forms": [
    { "person": "1sg", "past": "αγάπησα",   "present": "αγαπάω",  "future": "θα αγαπήσω" },
    { "person": "2sg", "past": "αγάπησες",  "present": "αγαπάς",  "future": "θα αγαπήσεις" },
    { "person": "3sg", "past": "αγάπησε",   "present": "αγαπάει", "future": "θα αγαπήσει" },
    { "person": "1pl", "past": "αγαπήσαμε", "present": "αγαπάμε", "future": "θα αγαπήσουμε" },
    { "person": "2pl", "past": "αγαπήσατε", "present": "αγαπάτε", "future": "θα αγαπήσετε" },
    { "person": "3pl", "past": "αγάπησαν",  "present": "αγαπάνε", "future": "θα αγαπήσουν" }
  ]
}
```

Возможные значения `person`: `1sg`, `2sg`, `3sg`, `1pl`, `2pl`, `3pl`.
Времена: `past` (аорист), `present` (настоящее), `future` (будущее с `θα`).

В файле сейчас десятки глаголов. Учитывай, что список будет расти —
не хардкодь количество, не предполагай порядок.

## Что должно уметь приложение

### 1. Режим «карточки» (главное)

- Случайно (или по порядку — на выбор пользователя) показывает по одному
  глаголу: сначала **только греческое слово** (`verb`), без перевода.
- По кнопке «Показать перевод» (или клику по карточке) открывается перевод
  (`translation`) и **таблица всех форм** (6 строк × 3 колонки: past / present /
  future, лица в порядке `1sg → 3pl`).
- Кнопки: **«Знаю»**, **«Не знаю»**, **«Дальше»** (или достаточно «Дальше» +
  «Показать»). Минимально: «Показать перевод» и «Следующее слово».
- Прогресс по сессии: счётчик «X из N» сверху.

### 2. Режим «список»

- Простой список всех глаголов: `verb — translation`.
- По клику разворачивается таблица форм для этого глагола (`<details>` подойдёт).
- Поле поиска сверху: фильтрация по `verb`, `translation`, `id` (подстрока,
  без учёта регистра, по всем формам тоже — чтобы можно было найти глагол
  по любой его форме).

### 3. Переключение режимов

- Сверху страницы две кнопки/таба: «Карточки» и «Список». Состояние можно
  не сохранять — достаточно переключения в рантайме.

## Опциональные приятные мелочи (только если просто)

- Запоминать в `localStorage` уже показанные/«знаю» слова, чтобы в режиме
  карточек они реже выпадали. Если усложняет — пропусти.
- Кнопка «Сбросить прогресс».

## Чего делать НЕ нужно

- Не делать красивый дизайн, не подключать шрифты, иконки, библиотеки UI.
- Не подключать никакие CDN.
- Не добавлять авторизацию, бэкенд, базу данных.
- Не трогать `docs/greek-words.json` и `docs/greek-words.md` — это источник
  правды, только чтение.
